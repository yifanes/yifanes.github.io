#io.Reader和io.Writer

##io.Writer

### 规范

1. writer方法向底层数据流写入len(p)字节的数据,这些数据来自于切片p
2. 返回被写入的字节数n, 0<= n <=len(p)
3. 如果n<len(p),则必须返回一些非nil的err
4. 如果中途出现问题,也要返回非nil的err
5. Writer方法绝对不可以修改切片p以及里面的数据



## io.Reader

###规范

1. Reader最多读取len(p)字节的数据,并保存到p
2. 返回读取的字节数以及任何发生的错误信息
3. n要满足 0 =< n <= len(p)
4. n < len(p)时,表示读取的数据不足以填充p,这时方法会立即返回,而不是等待更多的数据
5. 读取过程中遇到错误,会返回读取的字节数n以及相应的错误err
6. 在底层输入流结束时,方法会返回n>0的字节,但是err可能是EOF也可以是nil
7. 在第6种情况下,再次调用read方法的时候,肯定会返回0,eof
8. 调用Read方式时,如果n>0,优先处理读取的数据,然后再处理错误err,EOF也要这样处理
9. Read方法不鼓励返回n=0,并且err = nil的情况
